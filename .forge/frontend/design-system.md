## 1. Color System

### 1.1 Token Naming Convention

Plexica uses `--color-{scale}-{step}` naming for all palette tokens.
**NOT** the shadcn/ui convention (`--background`, `--foreground`, `--primary`, `--destructive`).

Tailwind classes map to CSS custom properties via `@plexica/ui/tailwind-preset`:
- `text-neutral-500` → `color: var(--color-neutral-500)`
- `bg-primary-600` → `background: var(--color-primary-600)`
- `text-error` → `color: var(--color-error)` (DEFAULT key)
- `bg-error-light` → `background: var(--color-error-light)`

## 2. Dark Mode

**Architecture**: CSS custom properties swapped via `[data-theme="dark"]` on `<html>`.

```html
<!-- Light mode (default) -->
<html>

<!-- Dark mode -->
<html data-theme="dark">
```

**Rule**: Do NOT use Tailwind `dark:` variant (e.g. `dark:bg-neutral-800`).
Dark mode works automatically via CSS variable values — use semantic Tailwind tokens
that reference CSS variables, and they will adapt automatically.

```tsx
// ✅ Correct — adapts to dark mode automatically
<div className="bg-neutral-100 text-neutral-900">

// ❌ Wrong — dark: variant not configured in Plexica's Tailwind
<div className="bg-white dark:bg-neutral-900">
```

## 4. Spacing Scale

Defined in `packages/ui/src/tokens/spacing.css`. Base unit: **4px**.
CSS variable naming: `--spacing-{N}` (not `--space-{N}`).

| CSS Token | Value | Tailwind class | Usage |
|-----------|-------|----------------|-------|
| `--spacing-0-5` | 2px | `p-0.5` | Tight inner padding |
| `--spacing-1` | 4px | `p-1` | Minimal gap (icon-text) |
| `--spacing-2` | 8px | `p-2` | Tight padding, gap between related items |
| `--spacing-3` | 12px | `p-3` | Input padding-x, form field gap |
| `--spacing-4` | 16px | `p-4` | Card padding, section gap |
| `--spacing-5` | 20px | `p-5` | Medium section padding |
| `--spacing-6` | 24px | `p-6` | Large content gap, page padding |
| `--spacing-8` | 32px | `p-8` | Section separator |
| `--spacing-10` | 40px | `p-10` | Page padding |
| `--spacing-12` | 48px | `p-12` | Large section separator |
| `--spacing-16` | 64px | `p-16` | Page top margin |
| `--spacing-20` | 80px | `p-20` | Hero spacing |
| `--spacing-24` | 96px | `p-24` | Max section spacing |

**Golden spacing rules**:
- Related elements gap: `gap-4` (16px)
- Section separator: `mt-8` or `space-y-8` (32px)
- Internal card padding: `p-4` or `p-6`
- Label → input gap: `mb-2` (8px)
- Button group gap: `gap-3` (12px)

## 8. Animation / Motion

| CSS Token | Value | Tailwind | Usage |
|-----------|-------|----------|-------|
| `--duration-fast` | `100ms` | (custom) | Toggles, button states |
| `--duration-normal` | `200ms` | `duration-200` | Dropdowns, tooltips, sidebar |
| `--duration-slow` | `300ms` | `duration-300` | Page transitions, modal open/close |

**Easing**: `cubic-bezier(0.4, 0, 0.2, 1)` (standard), `cubic-bezier(0.16, 1, 0.3, 1)` (enter).

**Motion accessibility**: Always wrap animations in:
```css
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; }
}
```

### 10.3 Component Composition Rules

1. **Error message**: `<p className="text-sm text-error" role="alert">` — never `text-red-*`, `text-destructive`
2. **Form field error**: `<p className="mt-1 text-sm text-error">{errors.field.message}</p>`
3. **Loading state**: `<SkeletonLoader variant="card" />` — never bare "Loading..." text
4. **Empty state**: `<EmptyState icon={...} heading={...} />` — never bare `<p>No data</p>`
5. **Confirmation dialog**: `<ConfirmDialog>` or `<DialogRoot>` — never `window.confirm()`
6. **Notifications**: `<Toast>` from @plexica/ui — never `sonner` (not installed)
7. **Icons**: `lucide-react` only — no emoji as icons, no custom SVG without design review

## 11. Golden Rules

1. **Never hardcode colors.** Use `text-neutral-*`, `text-primary-*`, `text-error`, `text-success`, `text-warning`, `text-info` or their bg/border variants.
2. **Never use shadcn/ui semantic tokens.** No `text-destructive`, `bg-muted`, `text-foreground`, `bg-background`, `ring-ring` — these don't exist in Plexica's Tailwind config.
3. **Never use raw Tailwind colors.** No `text-red-600`, `bg-green-100`, `text-yellow-800`.
4. **Never use dark: variant.** Dark mode works via CSS variable swapping, not Tailwind dark: classes.
5. **Never use arbitrary values.** No `text-[#dc2626]`, `p-[14px]`, `w-[calc(100%-2rem)]`.
6. **Always use spacing scale.** Every padding/margin/gap uses the defined spacing tokens.
7. **Always support keyboard.** Every interactive element is reachable via Tab + Enter/Esc.
