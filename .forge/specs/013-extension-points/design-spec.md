## 3. Component Specifications

### 3.1 Component: ExtensionSlot

**Type**: Container / Region
**Used on screens**: S1, S2, S3, S4
**FR coverage**: FR-007, FR-008, FR-009, FR-010, NFR-008, NFR-011, NFR-014

| Property | Value                                                                                   |
| -------- | --------------------------------------------------------------------------------------- |
| Variants | `action`, `panel`, `form`, `toolbar` (determined by slot `type`)                        |
| Layout   | action: horizontal flex-wrap, panel: vertical stack, form: tabbed, toolbar: inline flex |
| States   | empty, loading, loaded, partial-error, full-error, virtualized                          |

**Slot / Content:**

- `children`: Rendered `<ExtensionContribution>` wrappers in priority order
- `fallback`: React Suspense fallback (skeleton)

**Behavior:**

- On mount: query TanStack Query cache for contributions matching `slotId` + `workspaceId`. If cached (staleTime ≥ 60s), render immediately and revalidate in background.
- On load: show `<ExtensionSlotSkeleton>` per pending contribution.
- On success: replace skeleton with rendered contribution.
- On error (per contribution): show error fallback card for that contribution only; log via Pino.
- On contribution count > `virtualizationThreshold` (default: 20): switch to virtual scrolling with "Show N more extensions" button.
- On contributing plugin disabled: remove contribution from DOM without page reload (query cache invalidation via SSE).

**A11y:**

- Role: `role="region"` on outer container
- `aria-label`: `"Extensions: {slotLabel}"` (from slot declaration `label` field)
- `aria-busy="true"` while any contribution is loading
- Tab order: contributions are tabbable in priority order; each contribution is a tab stop
- Focus visible: standard `--ring` focus ring on interactive elements within contributions
- Screen reader: error fallback uses `role="alert"` for immediate announcement

### 3.2 Component: ExtensionContribution

**Type**: Wrapper / Error Boundary
**Used on screens**: S1, S2, S3, S4
**FR coverage**: FR-007, FR-009, US-007, NFR-008

| Property | Value                                                  |
| -------- | ------------------------------------------------------ |
| Variants | `loaded`, `loading`, `error`                           |
| Sizes    | Adapts to parent slot layout                           |
| States   | default, hover (shows extension badge), loading, error |

**Slot / Content:**

- `component`: The remote Module Federation component
- `contextProps`: Spread from slot's `contextSchema` + standard `PluginProps`
- `badge`: Subtle Puzzle icon (8px) at top-right corner, visible on hover

**Behavior:**

- On mount: React.lazy + Suspense loads the remote widget via Module Federation.
- On hover: subtle 8px `Puzzle` icon appears at the top-right corner with tooltip "[PluginName] extension".
- On error (React Error Boundary catch): render `<ExtensionErrorFallback>`.
- On 5-second load timeout: treat as error, show fallback.
- On keyboard Enter/Space: propagates to contributed component's own handlers.

**A11y:**

- Role: none (transparent wrapper; contributed component provides its own roles)
- Extension badge tooltip: `aria-label="Contributed by {pluginName}"`
- Badge is decorative by default; `aria-hidden="true"` on the icon, tooltip accessible via focus
- Keyboard: Tab enters the contributed component; Escape does nothing (no trap)

### 3.3 Component: ExtensionSlotSkeleton

**Type**: Loading Placeholder
**Used on screens**: S1, S2, S3, S4
**FR coverage**: FR-008, NFR-011

| Property | Value                                                                    |
| -------- | ------------------------------------------------------------------------ |
| Variants | `action-skeleton`, `panel-skeleton`, `form-skeleton`, `toolbar-skeleton` |
| Sizes    | Matches expected contribution dimensions per slot type                   |

**Layout per type:**

- `action-skeleton`: horizontal row of 2 pill-shaped skeletons (80×36px each)
- `panel-skeleton`: single card skeleton (full-width × 120px)
- `form-skeleton`: tab bar skeleton (3 tabs) + content area skeleton (full-width × 200px)
- `toolbar-skeleton`: inline row of 3 icon-button skeletons (32×32px each)

**Behavior:**

- Pulse animation using existing `Skeleton` component from `@plexica/ui`
- No layout shift: skeleton dimensions match rendered contribution dimensions
- Removed when contribution loads or timeout triggers error fallback

**A11y:**

- `aria-busy="true"` on the skeleton container
- `aria-label="Loading extension"` on the skeleton container
- Uses `--skeleton-from` / `--skeleton-to` tokens for animation

### 3.4 Component: ExtensionErrorFallback

**Type**: Alert / Feedback
**Used on screens**: S1, S2, S3, S4
**FR coverage**: US-007, NFR-008

| Property | Value                                                               |
| -------- | ------------------------------------------------------------------- |
| Variants | `compact` (for action/toolbar slots), `card` (for panel/form slots) |
| States   | default, dismissed                                                  |

**Slot / Content:**

- `compact`: inline text "Extension unavailable" with `AlertTriangle` icon (16px)
- `card`: Card with `AlertTriangle` icon (24px), title "Extension unavailable",
  body "[ContributionLabel] from [PluginName] could not load.", and [Dismiss] button

**Behavior:**

- On render: log structured error via Pino (`pluginId`, `slotId`, `contributionId`, `componentStack`, `tenantId`)
- On [Dismiss] click: remove the fallback card from the slot (contribution not retried until next page navigation)
- On keyboard Enter/Space on [Dismiss]: same as click

**A11y:**

- Role: `role="alert"` — announced immediately to screen readers
- `aria-live="assertive"` on the alert container
- [Dismiss] button: `aria-label="Dismiss failed extension from {pluginName}"`
- Keyboard: Tab focuses [Dismiss]; Enter/Space activates
- Color: `--error-boundary-icon` (warning amber) + text label; not color-only

### 3.5 Component: ExtensionSettingsPanel

**Type**: Settings Page / Accordion List
**Used on screens**: S5
**FR coverage**: FR-011, FR-022, FR-025, US-003, FR-033

| Property | Value                  |
| -------- | ---------------------- |
| Variants | N/A (single variant)   |
| States   | loaded, empty, loading |

**Structure:**

- Page heading: "Extensions" (h2)
- Subheading: "Control which cross-plugin extensions are visible in this workspace."
- Accordion groups: one per host plugin (using `PermissionGroupAccordion` pattern)
  - Group header: host plugin icon + name + badge with contribution count
  - Within each group: slot subheadings, then contribution rows

**Slot / Content:**

- Each contribution row contains:
  - Contributing plugin icon (16px) + name
  - Contribution label + type badge (`action` / `panel` / `form` / `toolbar`)
  - Description (truncated to 1 line, expandable)
  - Preview image thumbnail if `previewUrl` exists (FR-033), 80×60px
  - Toggle `Switch` (on/off)

**Behavior:**

- On toggle: `PATCH /api/v1/workspaces/:workspaceId/extension-visibility/:contributionId`
  with `{ isVisible: boolean }`. Auto-saves; no submit button needed.
- On toggle of tenant-disabled contribution: toggle is disabled + tooltip
  "Disabled by tenant admin. Contact your tenant administrator."
- On toggle of contribution with `validation_status: type_mismatch` or `target_not_found`:
  row shows warning badge `AlertTriangle` + tooltip explaining the issue. Toggle disabled.
- Accordion default state: all collapsed; most-recently-toggled group remembers open state via localStorage.

**A11y:**

- Accordion headers: `role="button"`, `aria-expanded="true|false"`, `aria-controls="panel-{id}"`
- Toggle: native `<Switch>` from `@plexica/ui` with `aria-label="Toggle {contributionLabel} visibility"`
- Disabled toggles: `aria-disabled="true"` + tooltip explains why
- Preview images: `alt="{contributionLabel} preview"` or `alt=""` if decorative
- Keyboard: Tab through accordion headers → Enter/Space to expand → Tab through toggles within
- Tab order: logical (header → first toggle → second toggle → ... → next header)

### 3.6 Component: ContributionRow

**Type**: List Item
**Used on screens**: S5
**FR coverage**: FR-022, FR-033

| Property | Value                                                         |
| -------- | ------------------------------------------------------------- |
| Variants | `enabled`, `disabled-tenant`, `disabled-workspace`, `warning` |
| States   | default, hover, toggling (optimistic), error                  |

**Layout:**

```
┌─────────────────────────────────────────────────────────┐
│ [Icon] Plugin Name  •  "Contribution Label"  [type]     │
│ Description text truncated to one line...      [Toggle] │
│ [Preview thumbnail 80×60]  (if previewUrl)              │
└─────────────────────────────────────────────────────────┘
```

**Behavior:**

- On hover: subtle background highlight (`--muted`)
- On toggle: optimistic UI — switch toggles immediately, reverts on PATCH failure with Toast error
- On error state: Toast "Failed to update extension visibility. Please try again."
- Warning variant: `AlertTriangle` icon + "Type mismatch" or "Target slot removed" badge

**A11y:**

- Entire row is not a single interactive element (toggle is the interactive part)
- Plugin name and label are read as static text
- Type badge: `aria-label="Type: {type}"`
- Warning badge: `aria-label="Warning: {validationMessage}"`

### 3.7 Component: ExtensionPermissionsPage

**Type**: Admin Settings Page
**Used on screens**: S6
**FR coverage**: FR-006, US-006

| Property | Value                  |
| -------- | ---------------------- |
| Variants | N/A (single variant)   |
| States   | loaded, empty, loading |

**Structure:**

- Page heading: "Extension Permissions" (h2)
- Subheading: "Control which plugins can contribute to other plugins' extension slots."
- Data table with columns: Plugin Name, Target Slot, Required Permission, Status, Action

**Table columns:**

| Column              | Width | Content                                                    |
| ------------------- | ----- | ---------------------------------------------------------- |
| Contributing Plugin | 25%   | Plugin icon + name                                         |
| Target Slot         | 25%   | "{HostPlugin} → {SlotLabel}" with type badge               |
| Required Permission | 25%   | Permission string in monospace font                        |
| Status              | 10%   | `Badge`: "Granted" (green) / "Not Granted" (gray)          |
| Action              | 15%   | `Button`: "Grant" (primary) / "Revoke" (destructive ghost) |

**Behavior:**

- On "Grant" click: confirmation dialog explaining scope ("This allows {pluginName} to
  contribute UI to {targetPlugin} views across all workspaces.")
- On "Revoke" click: confirmation dialog ("Revoking this permission will remove
  {pluginName}'s contributions from {targetPlugin} views in all workspaces.")
- On confirm: `POST /PATCH` API call. Table row updates with optimistic UI.
- Plugin disabled: entire row grayed out, action button replaced with "Plugin disabled"
  text.
- New plugin installed: row appears with "Not Granted" badge and a subtle highlight
  animation (yellow flash → fade) to draw attention.

**A11y:**

- Table: `<DataTable>` with sortable column headers, `aria-sort` attributes
- Grant/Revoke buttons: `aria-label="Grant extension permission for {pluginName}"` /
  `aria-label="Revoke extension permission for {pluginName}"`
- Status badges: `aria-label="Permission status: Granted"` / `"Permission status: Not Granted"`
- Confirmation dialog: `role="dialog"`, `aria-labelledby="confirm-title"`, focus trap, Esc to dismiss
- Keyboard: Tab through table rows → Tab to action button → Enter to trigger

### 3.8 Component: SlotInspectorOverlay

**Type**: Developer Tool Overlay
**Used on screens**: S7
**FR coverage**: FR-028, FR-031

| Property | Value                            |
| -------- | -------------------------------- |
| Variants | N/A (development mode only)      |
| States   | hidden, active (overlay visible) |

**Activation:** `Ctrl+Shift+E` key chord in development mode only (`NODE_ENV=development`).
Not available in production builds (tree-shaken by Vite).

**Layout:**

- Semi-transparent overlay highlighting all `<ExtensionSlot>` components on the page
- Each slot gets a colored outline (dashed `--primary` border, 2px) with a floating
  label showing slot metadata
- Floating label content: `slotId`, `type`, contribution count, host plugin name

**Behavior:**

- On activation: overlay appears; clicking a highlighted slot opens a detail panel
- Detail panel shows: slot ID, type, context schema (JSON), required permission,
  list of contributions (with priority, status, plugin name)
- On `Escape`: dismiss overlay
- On `Ctrl+Shift+E` again: toggle off

**A11y:**

- Developer tool only; reduced a11y requirements (not user-facing)
- Overlay: `role="dialog"`, `aria-label="Extension Slot Inspector"`
- Esc dismisses; focus trapped while open
- Detail panel scrollable with keyboard

### 3.9 Component: VirtualizedSlotContainer

**Type**: Virtualized List Container
**Used on screens**: S1, S2, S3, S4 (when contributions > threshold)
**FR coverage**: Edge Case #12

| Property  | Value                                                                 |
| --------- | --------------------------------------------------------------------- |
| Threshold | `virtualizationThreshold` prop (default: 20)                          |
| States    | collapsed (shows first 20 + "Show N more"), expanded (virtual scroll) |

**Behavior:**

- When contribution count ≤ threshold: render all contributions normally
- When count > threshold: render first 20, then a "Show {N} more extensions" button
- On "Show more" click: expand to full virtualized list with scroll
- Virtual scroll uses windowing to render only visible items (react-window or similar)

**A11y:**

- "Show N more" button: `aria-label="Show {N} more extensions in {slotLabel}"`
- Virtualized list: `role="list"`, items are `role="listitem"`
- Maintains focus position when scrolling
- Keyboard: arrow keys navigate within virtualized list

### 4.1 Wireframe: ExtensionSlot — Action Type (S1)

```
+------------------------------------------------------------------+
| HOST PLUGIN: Contact Detail View                    [Toolbar...] |
+------------------------------------------------------------------+
|                                                                  |
|  Contact: John Smith                                             |
|  Email: john@example.com                                         |
|  Phone: +1-555-0123                                              |
|                                                                  |
|  +--- Extensions: Contact Actions ---- role="region" ----------+ |
|  |                                                              | |
|  |  [📧 Send Email]  [📄 Create Invoice ⧉]  [📊 Engagement ⧉] | |
|  |                       (Billing)              (Analytics)     | |
|  |                                                              | |
|  +--------------------------------------------------------------+ |
|                                                                  |
|  [Other host content continues below...]                         |
+------------------------------------------------------------------+

⧉ = subtle Puzzle icon (8px), visible on hover only
```

**States:**

- **Default**: Contributions rendered as `Button variant="outline"` in horizontal flex-wrap
- **Loading**: `┌──▓▓▓▓▓▓──┐ ┌──▓▓▓▓▓▓──┐` (pill-shaped skeletons, 80×36px each)
- **Error (single contribution)**: `[📄 Create Invoice]  [⚠ Extension unavailable]  [📊 Engagement]`
  — compact inline error replaces failed contribution
- **Error (all contributions fail)**: Slot region shows "Extensions could not load. [Retry]"
- **Empty (no contributions)**: Slot region collapses to zero height (not visible to users)
- **Success**: Skeletons fade out, buttons fade in (transition: `--transition-fast`)

**Accessibility (S1):**

- `role="region"` with `aria-label="Extensions: Contact Actions"`
- Tab order: [Send Email] → [Create Invoice] → [Engagement]
- Each button: `aria-label` includes plugin origin on focus (e.g., "Create Invoice, from Billing plugin")
- Error fallback: `role="alert"`, `aria-live="assertive"`
- Loading: `aria-busy="true"`, `aria-label="Loading extensions"`

### 4.6 Wireframe: Tenant Extension Permissions (S6)

**States:**

- **Default**: DataTable with sortable columns
- **Loading**: Skeleton table (4 rows with pulsing cells)
- **Empty**: EmptyState: "No extension permissions to manage. Permissions appear
  when plugins declare contributions to other plugins' slots."
- **Grant confirmation**: Dialog: "Grant extension permission? This allows
  [PluginName] to contribute UI to [TargetPlugin] views across all workspaces.
  Workspace admins can further control visibility per workspace."
  — [Cancel] [Grant Permission]
- **Revoke confirmation**: Dialog: "Revoke extension permission? This will remove
  [PluginName]'s contributions from [TargetPlugin] views in all workspaces."
  — [Cancel] [Revoke Permission] (destructive variant)
- **New plugin highlight**: Row appears with brief yellow flash animation
  (`@keyframes highlight { 0% { bg: --status-warning/10% } 100% { bg: transparent } }`,
  2s duration)

**Accessibility (S6):**

- Table: `<DataTable>` from `@plexica/ui`; `aria-sort` on sortable columns
- Status badge: `aria-label="Permission status: {status}"`
- Grant button: `aria-label="Grant extension permission for {pluginName}"`
- Revoke button: `aria-label="Revoke extension permission for {pluginName}"`
- Disabled row: `aria-disabled="true"` on the row; "Plugin disabled" text
- Confirmation dialogs: `role="dialog"`, `aria-labelledby`, focus trap, Esc dismiss
- Keyboard: Tab through table → Tab to action button → Enter to open dialog → Tab within dialog
- Permission strings: `<code>` with `font-mono` for readability

**Responsive (S6):**

- ≥1024px: full table layout
- 768–1023px: "Required Permission" column hidden; accessible via row expansion
- <768px: not primary target (admin desktop-first)
