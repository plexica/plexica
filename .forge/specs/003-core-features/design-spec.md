## 3. Wireframes

### Global Navigation Pattern — WorkspaceSelectorDropdown

**Source**: FR-001, FR-003, FR-004 | **Present on**: All authenticated screens

The workspace selector is a dropdown control embedded in the top navigation bar
between the app logo and the utility icons (search, notifications, user menu).
It replaces the previous sidebar workspace tree. The label reflects the currently
active workspace; when at tenant level (no workspace context), it shows
`Workspaces`.

#### Top Nav — closed state (workspace context)

```
+------------------------------------------------------------------------+  W: 1440px
| [≡] Plexica    [Frontend Team ▾]                    [🔍] [🔔 3] [AV ▾] |
+------------------------------------------------------------------------+
```

#### Top Nav — closed state (tenant level, no active workspace)

```
+------------------------------------------------------------------------+  W: 1440px
| [≡] Plexica    [Workspaces ▾]                       [🔍] [🔔 3] [AV ▾] |
+------------------------------------------------------------------------+
```

#### WorkspaceSelectorDropdown — open state

```
+------------------------------------------------------------------------+
| [≡] Plexica    [Frontend Team ▾]↓                   [🔍] [🔔 3] [AV ▾] |
+------------------------------------------------------------------------+
         +------------------------------------------+
         | [🔍 Search workspaces..._______________]  |
         +------------------------------------------+
         | [▼] Engineering                          |
         |   [▼] Frontend Team        [● active]   |
         |       [ ] Sprint Planning               |
         |       [ ] Retrospective                 |
         |   [ ] Backend Team                      |
         |   [ ] DevOps                            |
         | [▶] Marketing                           |
         | [ ] HR                                  |
         +------------------------------------------+
         | [≡] View all workspaces              →  |
         +------------------------------------------+
```

`[● active]` marks the currently selected workspace.
`View all workspaces →` navigates to the Workspace List page (Screen 1).

#### State Inventory

| State        | Trigger                       | Visual Change                                          | User Feedback                      |
| ------------ | ----------------------------- | ------------------------------------------------------ | ---------------------------------- |
| Closed       | Default                       | Button shows active workspace name or "Workspaces"     | —                                  |
| Open         | Click / Enter / Space         | Dropdown panel with search + tree appears              | —                                  |
| Searching    | User types in search field    | Tree filters to matching workspaces (non-match hidden) | aria-live "N results"              |
| Loading tree | Dropdown opens, fetch pending | Skeleton tree items inside panel                       | aria-busy="true"                   |
| Selected     | Click or Enter on tree item   | Panel closes, label updates, app navigates             | —                                  |
| Empty search | No workspace matches query    | "No workspaces found." message                         | "No workspaces match your search." |

#### Interactive Elements

| Element             | Type           | Action      | Outcome                                          | FR Ref |
| ------------------- | -------------- | ----------- | ------------------------------------------------ | ------ |
| Selector trigger    | Button (ghost) | Click/Enter | Opens dropdown panel                             | FR-004 |
| Search input        | Input (text)   | Type        | Filters visible tree nodes (debounced 200ms)     | FR-001 |
| Tree node (parent)  | Button (ghost) | Click/Enter | Expands / collapses children                     | FR-004 |
| Tree node (leaf)    | Button (nav)   | Click/Enter | Selects workspace, navigates to Workspace Detail | FR-003 |
| View all workspaces | Link           | Click/Enter | Navigates to Workspace List page (Screen 1)      | FR-001 |

#### Accessibility

```
Role: button (trigger) with aria-haspopup="listbox", aria-expanded="true|false"
aria-label on trigger: "Select workspace: [current name]" or "Select workspace: Workspaces"
Dropdown panel: role="dialog" when open; focus moves to search input on open
Search input: aria-label="Search workspaces"
Tree: role="tree" inside panel; each item role="treeitem"
Tree expand/collapse: aria-expanded="true|false"
Active item: aria-selected="true" + visual [● active] marker
"View all workspaces" link: role="link", positioned last in tab order within panel
Focus trap: Yes — Tab cycles within open panel; Esc closes and returns focus to trigger
Live region: aria-live="polite" on search results count

Keyboard:
  Enter/Space on trigger: open/close panel
  Esc: close panel, return focus to trigger
  Arrow Up/Down in tree: navigate visible items
  Arrow Right: expand collapsed item / descend to first child
  Arrow Left: collapse expanded item / move to parent
  Enter/Space on item: select workspace and close panel
  Tab: cycles from search → tree nodes → "View all" link → back to search
```

#### Responsive Notes

- **1440px**: Full layout — navigation sidebar (260px) + content area with table columns: Name (hierarchy-indented), Role badge, Members count, Last activity timestamp. Workspace selector visible in top nav.

- **768px**: Sidebar hidden (hamburger toggle). Table shows: Name, Role, Members. Last activity hidden.
- **375px**: Sidebar hidden. Full-width card layout replaces table. Each card: workspace name, role badge, member count. Tap opens workspace detail.
