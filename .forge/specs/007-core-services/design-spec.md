### Design Decisions Summary

1. **Header density**: Search bar, notification bell, language selector, and user avatar share the header. Search collapses to an icon on screens < 768px. Bell and language selector remain icon-only at all sizes. Per compact header concern raised during discovery.
2. **Real-time updates**: Notification badge count and job status counters update via Server-Sent Events (SSE). UI must handle live state transitions without full page reload. Per real-time updates constraint.
3. **Component reuse**: Leverages existing `DataTable`, `Badge`, `Progress`, `Toast`, `EmptyState`, `Skeleton`, `Spinner`, `Tooltip`, `Tabs`, and `Card` components. 6 new components introduced.

#### State Inventory

| State          | Trigger                            | Visual Change                                                                                       | User Feedback                             |
| -------------- | ---------------------------------- | --------------------------------------------------------------------------------------------------- | ----------------------------------------- |
| Default (idle) | Search bar focused, no input       | Show recent searches (last 5) or "Start typing to search..."                                        | —                                         |
| Loading        | User typing (after 300ms debounce) | Skeleton placeholders (3 rows) with pulsing animation                                               | —                                         |

| No Results     | Search API returns empty           | `EmptyState` with Search icon: "No results for '[query]'"                                           | Suggestion: "Try a different search term" |
| Error          | Search API fails (500/timeout)     | `Alert` variant destructive: "Search is temporarily unavailable. Please try again." with Retry link | —                                         |

#### Interactive Elements

| Element               | Type                 | Action                         | Outcome                                 | FR Ref |
| --------------------- | -------------------- | ------------------------------ | --------------------------------------- | ------ |
| Search input          | text input           | Type query                     | Triggers debounced search after 300ms   | FR-011 |

| Result row            | link (clickable div) | Click                          | Navigates to resource detail page       | FR-013 |
| Type group header     | heading              | —                              | Non-interactive; groups results by type | FR-014 |
| "See all results"     | link                 | Click                          | Navigates to full search results page   | FR-013 |
| `/` keyboard shortcut | keyboard             | Press `/` when not in an input | Opens search overlay with focus         | —      |
| Arrow keys ↑↓         | keyboard             | Navigate results               | Moves focus between result rows         | —      |
| Enter                 | keyboard             | Select focused result          | Navigates to that result                | —      |
| Esc                   | keyboard             | Close overlay                  | Returns focus to previous element       | —      |

#### Responsive Notes

- **1440px**: Full-width search input in header (300px). Overlay is 640px wide, centered in content area.
- **1024px**: Search input shrinks to 200px. Overlay is 560px wide.
- **768px**: Search input collapses to icon-only (🔍). Click opens full-width overlay below header.
- **375px**: Search icon in header. Overlay takes full screen width. Results show truncated metadata.

#### Accessibility

```
Role: dialog (search overlay is role="dialog" with aria-label="Search")
Heading hierarchy: — (no h-tags inside overlay; type groups use aria-label)
Tab order: [1] Search input → [2] First result → [3..N] Subsequent results → [N+1] "See all results"
Focus trap: Yes — when overlay is open, Tab cycles within overlay. Esc closes.

ARIA:
  Search input: role="combobox", aria-expanded="true", aria-controls="search-results",
                aria-activedescendant="result-{id}" (updates with arrow key navigation)
  Results list: role="listbox", id="search-results"
  Each result: role="option", id="result-{id}"
  Type group header: role="presentation" (purely visual grouping)
  Result count: aria-live="polite" — announces "N results found" after search completes
  Close button: aria-label="Close search"

Contrast:
  Body text on overlay bg: --foreground on --card = 17.4:1 ✓
  Muted text (metadata): --muted-foreground on --card = 4.7:1 ✓
  Highlighted match text: --primary on --card = 5.3:1 ✓

Keyboard:
  `/`: Opens search overlay (when no input is focused)
  Esc: Closes search overlay, returns focus to header search trigger
  ↑↓: Navigate between results (aria-activedescendant updates)
  Enter: Activate selected result (navigate to page)
  Tab: Move between search input → results → "See all" link

Screen reader flow:
  1. "Search dialog" (dialog role announcement)
  2. "Search across all data, combobox" (input role)
  3. "6 results found" (aria-live polite, after results load)
  4. "Contacts group, 3 results" (when navigating into group)
  5. "John Smith, Acme Sales, option 1 of 6" (result announcement)
```
