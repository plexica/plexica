### Journey 1: Maria — Configure Workspace Settings

**Trigger**: Maria has created a new workspace for her Design team and needs to configure sharing, member limits, and discoverability before adding team members.

**Preconditions**:

- Maria is authenticated and has tenant context
- Maria is ADMIN of the "design-team" workspace
- Workspace has default settings (all defaults from schema)

#### Happy Path

| Step | User Action                                         | System Response                                                                                                         | Emotional State                              |
| ---- | --------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- | -------------------------------------------- |
| 1    | Maria opens the WorkspaceSwitcher dropdown          | Dropdown shows all her workspaces with names, slugs, and member counts. "design-team" is listed                         | 😐 Neutral                                   |
| 2    | Maria selects "design-team" workspace               | WorkspaceContext updates, header shows "Design Team" as active workspace, all API calls now scoped                      | 😐 Neutral                                   |
| 3    | Maria navigates to Workspace Settings page          | Settings page loads with current values in a structured form: toggles, selects, number inputs. All fields show defaults | 😊 Relieved — sees a real form, not raw JSON |
| 4    | Maria toggles "Allow Cross-Workspace Sharing" to ON | Toggle animates to ON state. No save yet — unsaved indicator appears                                                    | 😐 Neutral                                   |
| 5    | Maria changes "Maximum Members" to 25               | Number input updates. Unsaved indicator persists                                                                        | 😐 Neutral                                   |
| 6    | Maria sets "Default Team Role" to ADMIN             | Select dropdown shows ADMIN selected                                                                                    | 🤔 Uncertain — is ADMIN the right default?   |
| 7    | Maria changes "Default Team Role" back to MEMBER    | Select shows MEMBER. She feels more confident with a safer default                                                      | 😊 Relieved                                  |
| 8    | Maria clicks "Save Settings"                        | Loading spinner on button. After ~300ms, success toast: "Workspace settings saved". Unsaved indicator disappears        | 🎉 Satisfied                                 |

**Outcome**: Workspace settings are persisted with typed validation. Maria is confident the settings are enforced because the form prevented invalid values.

**Delight opportunities**:

- Step 3: Show descriptions under each setting explaining its effect in plain language
- Step 4: Show an inline hint: "Enabling sharing allows other workspace admins to share plugins with this workspace"
- Step 8: Highlight which settings changed from their previous values in the success feedback

---

#### Edge Case A: Validation Error on Save

**Trigger**: Maria enters a negative number in the "Maximum Members" field and tries to save.

| Step | User Action                                  | System Response                                                                                                                    | Emotional State        |
| ---- | -------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- | ---------------------- |
| 1-5  | Same as happy path through toggling settings | Form shows current values                                                                                                          | 😐 Neutral             |
| 6    | Maria types "-5" in "Maximum Members"        | Client-side validation fires immediately: red border on input, inline error "Must be 0 or greater (0 = unlimited)" below the field | 😤 Slightly frustrated |
| 7    | Maria changes value to "25"                  | Red border clears, error message disappears, Save button re-enables                                                                | 😊 Relieved            |
| 8    | Maria clicks "Save Settings"                 | Success toast appears                                                                                                              | 🎉 Satisfied           |

**Recovery path**: Client-side validation prevents invalid submissions. Save button is disabled while any field has validation errors. Error messages are specific and actionable.
**Feedback**: "Must be 0 or greater (0 = unlimited)" — explains both the constraint and the special value.

#### Edge Case C: Server Error on Save

**Trigger**: API returns 500 when saving settings due to a transient database error.

| Step | User Action                                     | System Response                                                                                                                                                 | Emotional State |
| ---- | ----------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------- |
| 1-7  | Same as happy path through configuring settings | Form filled with desired values                                                                                                                                 | 😐 Neutral      |
| 8    | Maria clicks "Save Settings"                    | Button shows spinner for ~2s, then error toast: "Failed to save settings. Your changes have been preserved — please try again." Form retains all entered values | 😤 Frustrated   |
| 9    | Maria clicks "Save Settings" again              | Button shows spinner, after ~300ms success toast: "Workspace settings saved"                                                                                    | 😊 Relieved     |

**Recovery path**: Form values are preserved on error. User can retry immediately without re-entering data. Error toast auto-dismisses after 8 seconds.
**Feedback**: "Failed to save settings. Your changes have been preserved — please try again."

### Journey 2: Kenji — Switch Workspace and Browse Members

**Trigger**: Kenji needs to switch from the "backend" workspace to the "platform" workspace to check who is on the team before adding a cross-team collaborator.

**Preconditions**:

- Kenji is authenticated with tenant context
- Kenji is MEMBER of "backend" (currently active) and MEMBER of "platform"
- "platform" workspace has 23 members

#### Happy Path

| Step | User Action                                                | System Response                                                                                                         | Emotional State                                      |
| ---- | ---------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------- |
| 1    | Kenji presses keyboard shortcut (Cmd+K or clicks switcher) | WorkspaceSwitcher dropdown opens with focus on search field                                                             | 😐 Neutral                                           |
| 2    | Kenji types "plat" in search field                         | Workspace list filters to show "Platform Team" matching the query. Other workspaces are hidden                          | 😊 Efficient — found it quickly                      |
| 3    | Kenji presses Enter to select "Platform Team"              | Dropdown closes. Header updates to "Platform Team". WorkspaceContext fetches new workspace data. Page content refreshes | 😐 Neutral                                           |
| 4    | Kenji navigates to Members page                            | DataTable shows 23 members with columns: Name, Email, Role, Joined. Role badges (ADMIN/MEMBER/VIEWER) color-coded       | 😐 Neutral                                           |
| 5    | Kenji uses Role filter dropdown to select "ADMIN"          | Table filters to show only ADMIN members (3 results). Result count updates: "3 of 23 members"                           | 😊 Found the admins quickly                          |
| 6    | Kenji notes the admin names and switches back to "backend" | Opens switcher, types "back", selects "Backend Team". Context switches back                                             | 🎉 Satisfied — got the info he needed in <30 seconds |

**Outcome**: Kenji identified the admins of the Platform workspace in under 30 seconds, enabling him to request collaborator access.

**Delight opportunities**:

- Step 1: Show recently-used workspaces at the top of the dropdown for fast switching
- Step 4: Show Kenji's own role as a subtle "Your role: MEMBER" badge at the top of the members list
- Step 5: Show role distribution summary: "3 Admins, 15 Members, 5 Viewers"

### Journey 3: Maria — Share a Plugin Across Workspaces

**Trigger**: Maria wants to share the "Analytics Dashboard" plugin from the "engineering" workspace to the "design-team" workspace so the design team can access analytics without a separate plugin installation.

**Preconditions**:

- Maria is ADMIN of both "engineering" and "design-team" workspaces
- "Analytics Dashboard" plugin is installed in "engineering" workspace
- "design-team" workspace has `allowCrossWorkspaceSharing: true` in settings
- "engineering" workspace has `allowCrossWorkspaceSharing: true` in settings

#### Happy Path

| Step | User Action                                                                    | System Response                                                                                                                                                                     | Emotional State                        |
| ---- | ------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------- |
| 1    | Maria navigates to "engineering" workspace and opens the Shared Resources page | Page shows a DataTable of currently shared resources (may be empty). Header: "Shared Resources". Subheading: "Share plugins with other workspaces in your organization"             | 😐 Neutral                             |
| 2    | Maria clicks "Share Resource" button                                           | Share Resource dialog opens. Dialog contains: resource type selector (currently only "Plugin"), a searchable dropdown to select the resource, and a confirmation section            | 😐 Neutral                             |
| 3    | Maria selects resource type "Plugin"                                           | Plugin dropdown loads with installed plugins in this workspace                                                                                                                      | 😐 Neutral                             |
| 4    | Maria selects "Analytics Dashboard" from plugin dropdown                       | Plugin name, version, and description shown in a preview card below the dropdown                                                                                                    | 😊 Confident — correct plugin selected |
| 5    | Maria clicks "Share" button in the dialog                                      | Loading state on button. After ~300ms, success toast: "Analytics Dashboard shared from Engineering workspace". Dialog closes. DataTable updates to show the new shared resource row | 🎉 Satisfied                           |
| 6    | Maria switches to "design-team" workspace and navigates to Shared Resources    | Page shows "Analytics Dashboard" listed as a shared resource with source workspace "Engineering" and share date                                                                     | 😊 Confirmed — sharing works both ways |

**Outcome**: The Analytics Dashboard plugin is now accessible in the "design-team" workspace without reinstallation. Maria confirmed the share from both sides.

**Delight opportunities**:

- Step 1: Show a count badge on the "Shared Resources" nav item showing how many resources are shared
- Step 4: Show a preview of the plugin's icon and description to prevent sharing the wrong plugin
- Step 6: In the target workspace, show the source workspace name as a clickable link (if user has access)

#### Edge Case B: Resource Already Shared

**Trigger**: Maria tries to share a plugin that is already shared.

| Step | User Action                                          | System Response                                                                                                                                            | Emotional State                |
| ---- | ---------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------ |

- Priya is VIEWER in "engineering", "design-team", and "marketing" workspaces

#### Happy Path

| Step | User Action                                           | System Response                                                                                                                                                 | Emotional State                           |
| ---- | ----------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------- |

| 3    | Priya navigates to Members page                       | DataTable shows 45 members. Columns: Name, Email, Role, Joined Date. Role badges are color-coded. At 150% zoom, table remains readable — no horizontal overflow | 😊 Relieved — zoom doesn't break layout   |
