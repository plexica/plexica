### Scope

This design spec covers the **3 user-facing phases** of Spec 010:

1. **Phase 1: Error Boundaries** — Plugin-level and root-level error fallback UIs
2. **Phase 2: Tenant Theming** — Themed shell (header logo, colors, fonts)
3. **Phase 3: Widget System** — Widget loading skeleton, rendered widget, unavailable fallback

Phases 4 (Test Coverage) and 5 (Accessibility Audit) are developer/QA-focused
with no new user-facing screens and are not wireframed here.

## 3. Wireframes

### Screen 1: Plugin Error Fallback

**Source**: FR-016, FR-017, FR-018 | **Type**: New

Per FR-016, this boundary wraps every plugin route. Per FR-017, the fallback
shows the plugin name, a user-friendly message, the error summary (no stack
trace), and [Retry] + [Go Back] buttons. Per FR-018, structured error context
is logged to console.

**Desktop (1440px)**:

```
+--------------------------------------------------------------------+
| [Tenant Logo]  Dashboard  CRM  Analytics       [Bell] [User ▾]    |
+--------------------------------------------------------------------+
| ┌────────────┐                                                     |
| │ Dashboard  │                                                     |
| │ CRM     *  │   +-----------------------------------------+      |
| │ Analytics  │   |                                         |      |
| │ Settings   │   |        /!\                              |      |
| │            │   |    Plugin Unavailable                   |      |
| │            │   |                                         |      |
| │            │   |  The "CRM" plugin could not be loaded.  |      |
| │            │   |  This might be a temporary issue.       |      |
| │            │   |                                         |      |
| │            │   |  +-----------------------------------+  |      |
| │            │   |  | Failed to fetch remote entry      |  |      |
| │            │   |  +-----------------------------------+  |      |
| │            │   |                                         |      |
| │            │   |     [ Retry ]    [ Go Back ]            |      |
| │            │   |                                         |      |
| │            │   +-----------------------------------------+      |
| │            │                                                     |
| └────────────┘                                                     |
+--------------------------------------------------------------------+
```

**Notes**:

- Sidebar and header remain fully functional (error is contained)
- `*` indicates active nav item
- Warning icon: `AlertTriangle` from Lucide
- Error message box: `--muted` background, `--font-mono` font, `--font-size-sm`
- Card: `--card` background, `--shadow-md`, `--radius-lg`, centered in main content area
- Min-height: 400px for the main content area to center the card vertically

**Mobile (375px)**:

```
+----------------------+
| [=] [Logo]    [Bell] |
+----------------------+
|                      |
|       /!\            |
|  Plugin Unavailable  |
|                      |
|  The "CRM" plugin    |
|  could not be loaded.|
|  This might be a     |
|  temporary issue.    |
|                      |
|  +----------------+  |
|  | Failed to fetch|  |
|  | remote entry   |  |
|  +----------------+  |
|                      |
|  [    Retry     ]    |
|  [   Go Back    ]    |
|                      |
+----------------------+
```

**Notes**:

- Sidebar collapsed to hamburger menu (`[=]`)
- Buttons stack vertically, full-width
- Card uses full width with `--space-4` horizontal padding

#### State Inventory

| State      | Trigger                  | Visual Change                                                    | User Feedback                        |
| ---------- | ------------------------ | ---------------------------------------------------------------- | ------------------------------------ |
| Default    | Error boundary activates | Centered card with warning icon, message, error, action buttons  | "Plugin Unavailable"                 |
| Retry      | User clicks [Retry]      | Error boundary resets; children re-mount; Suspense shows loading | Loading spinner in main content area |
| Retry Fail | Re-mount throws again    | Error boundary re-activates; same fallback shown                 | Same message appears again           |
| Go Back    | User clicks [Go Back]    | Browser navigates to previous page                               | Previous page renders                |

#### Interactive Elements

| Element   | Type   | Action | Outcome                                               |
| --------- | ------ | ------ | ----------------------------------------------------- |
| [Retry]   | Button | Click  | Resets error boundary; re-attempts plugin lazy import |
| [Go Back] | Button | Click  | Calls `navigate(-1)` via TanStack Router              |

#### Responsive Notes

- **1440px**: Card is `max-w-md` (28rem / 448px), centered horizontally and vertically in the main content area. Sidebar visible.
- **768px**: Same card layout. Sidebar may be collapsed depending on user preference.
- **375px**: Card fills available width with `--space-4` padding. Buttons stack vertically. Sidebar hidden behind hamburger menu.

#### Accessibility

```
Role: main (landmark for main content area)
Heading hierarchy: h2 "Plugin Unavailable"
Tab order: [1] Retry button -> [2] Go Back button
Focus trap: No (not a modal — user can still interact with sidebar/header)

ARIA:
  Fallback card container: role="alert" (announces to screen reader on appearance)
  Warning icon: aria-hidden="true" (decorative; heading conveys meaning)
  Error message box: role="status" aria-live="polite"
  [Retry] button: aria-label="Retry loading CRM plugin"
  [Go Back] button: aria-label="Go back to previous page"

Contrast:
  Heading text on card: --foreground / --card = 17.4:1 (light), 17.4:1 (dark) ✓
  Error text on muted bg: --foreground / --muted = 14.2:1 ✓
  Button text on primary: --primary-foreground / --primary = 5.3:1 ✓

Keyboard:
  Tab: moves focus between [Retry] and [Go Back]
  Enter / Space: activates focused button
  No Esc binding (not a modal)

Screen reader flow:
  1. Main landmark announced
  2. "alert: Plugin Unavailable"
  3. "The CRM plugin could not be loaded. This might be a temporary issue."
  4. "status: Failed to fetch remote entry"
  5. "button: Retry loading CRM plugin"
  6. "button: Go back to previous page"
```

### Screen 2: Root Error Page

**Source**: FR-016, Edge Case #11 | **Type**: New

The root error boundary catches errors in the shell itself (Router, AuthProvider,
ThemeProvider). Because the shell component tree is broken, this is a **full-page
layout** with no sidebar or header. It includes the Plexica wordmark logo (not
tenant logo — ThemeProvider may be the source of the crash), an apologetic
message, a [Reload Page] button, and support contact information.

**Desktop (1440px)**:

```
+--------------------------------------------------------------------+
|                                                                    |
|                                                                    |
|                       [Plexica Logo]                               |
|                                                                    |
|                    Something Went Wrong                            |
|                                                                    |
|         We're sorry — something unexpected happened.               |
|         Please try reloading the page.                             |
|                                                                    |
|                    [ Reload Page ]                                  |
|                                                                    |
|         If this keeps happening, contact support at                |
|         support@plexica.io                                         |
|                                                                    |
|                                                                    |
+--------------------------------------------------------------------+
```

**Notes**:

- Full-viewport centered layout (no sidebar, no header)
- Plexica wordmark logo is a static SVG (`/plexica-logo.svg`) — not tenant logo
- Background: `--background`
- Max-width of content block: `max-w-lg` (32rem / 512px)
- [Reload Page] calls `window.location.reload()`
- Support email is a `mailto:` link

**Mobile (375px)**:

```
+----------------------+
|                      |
|   [Plexica Logo]     |
|                      |
|   Something Went     |
|   Wrong              |
|                      |
|   We're sorry --     |
|   something          |
|   unexpected         |
|   happened. Please   |
|   try reloading.     |
|                      |
|  [ Reload Page ]     |
|                      |
|   If this keeps      |
|   happening, contact |
|   support@plexica.io |
|                      |
+----------------------+
```

#### State Inventory

| State      | Trigger                   | Visual Change                              | User Feedback                           |
| ---------- | ------------------------- | ------------------------------------------ | --------------------------------------- |
| Default    | Root error caught         | Full-page error layout                     | "Something Went Wrong"                  |
| Reload     | User clicks [Reload Page] | Full page refresh                          | Browser loading indicator               |
| Persistent | Error recurs after reload | Same error page; support info is prominent | "contact support at support@plexica.io" |

#### Interactive Elements

| Element       | Type   | Action | Outcome                           |
| ------------- | ------ | ------ | --------------------------------- |
| [Reload Page] | Button | Click  | `window.location.reload()`        |
| support@...   | Link   | Click  | Opens `mailto:support@plexica.io` |

#### Responsive Notes

- **1440px**: Content centered both horizontally and vertically. Logo 48px height.
- **768px**: Same layout, logo 40px.
- **375px**: Same layout, logo 32px. Text reflows naturally. Button full-width.

#### Accessibility

```
Role: main (the entire page is the main landmark)
Heading hierarchy: h1 "Something Went Wrong"
Tab order: [1] Reload Page button -> [2] support@plexica.io link
Focus trap: No

ARIA:
  Container: role="alert" (announces immediately to screen reader)
  Logo: alt="Plexica" (informative)
  [Reload Page]: aria-label="Reload this page"
  support link: standard mailto link (no extra ARIA needed)

Contrast:
  Heading on background: --foreground / --background = 17.4:1 ✓
  Body text on background: --foreground / --background = 17.4:1 ✓
  Support text (muted): --muted-foreground / --background = 4.7:1 ✓
  Button: --primary-foreground / --primary = 5.3:1 ✓

Keyboard:
  Tab: moves between [Reload Page] and [support link]
  Enter / Space: activates focused element

Screen reader flow:
  1. "alert: Something Went Wrong"
  2. "We're sorry — something unexpected happened. Please try reloading the page."
  3. "button: Reload this page"
  4. "If this keeps happening, contact support at support@plexica.io"
  5. "link: support@plexica.io"
```

### Screen 3: Themed Shell — Header

**Source**: FR-019, FR-020, FR-021 | **Type**: Modified

The shell header is modified to display the tenant logo (FR-021) and apply
tenant brand colors (FR-020). The logo appears in the top-left corner, is
clickable (links to `/`), and falls back to the Plexica wordmark if no tenant
logo is configured or if the image fails to load.

**Desktop (1440px)**:

```
+--------------------------------------------------------------------+
| [Tenant Logo]  Dashboard  CRM  Analytics       [Bell] [Globe] [AV]|
+--------------------------------------------------------------------+
   ^                                                         ^    ^
   |                                                         |    |
   Logo area: max-h 40px, max-w 200px                   Lang  Avatar
   Clickable -> navigates to "/"                        selector menu
   Fallback: Plexica wordmark SVG
```

**With Default Theme (no tenant logo)**:

```
+--------------------------------------------------------------------+
| [Plexica]  Dashboard  CRM  Analytics            [Bell] [Globe] [AV]|
+--------------------------------------------------------------------+
```

**With Tenant Theme (custom logo + colors)**:

```
+--------------------------------------------------------------------+  --color-surface bg
| [ACME Logo]  Dashboard  CRM  Analytics          [Bell] [Globe] [AV]|
+--------------------------------------------------------------------+
                         ^
                         Active nav item underline uses --color-primary
```

**Mobile (375px)**:

```
+----------------------+
| [=] [Logo]     [AV]  |
+----------------------+
  ^      ^
  |      |
  Hamburger  Logo: max-h 32px
  opens sidebar overlay
```

**Notes**:

- Header height: `--header-height` (56px)
- Background: `--card` (light: `#FFFFFF`, dark: `#111111`)
- Border-bottom: 1px solid `--border`
- Logo constraints: `max-h-10` (40px desktop), `max-h-8` (32px mobile), `max-w-[200px]`, `object-contain`
- Logo `onError` handler swaps `src` to `/plexica-logo.svg`
- Navigation items use `--foreground` text; active item has `--primary` bottom border (2px)
- `[Bell]` — NotificationBell component (Spec 007)
- `[Globe]` — LanguageSelector component (Spec 006)
- `[AV]` — Avatar dropdown with user menu

#### State Inventory

| State         | Trigger                        | Visual Change                                               | User Feedback            |
| ------------- | ------------------------------ | ----------------------------------------------------------- | ------------------------ |
| Default       | Theme loaded                   | Tenant logo + brand colors applied                          | Branded header           |
| Theme Loading | ThemeProvider fetching         | Default theme colors applied; no flash (preloaded defaults) | Plexica defaults shown   |
| No Logo       | `theme.logo` is null/undefined | Plexica wordmark shown instead of tenant logo               | —                        |
| Broken Logo   | `<img>` onError fires          | Plexica wordmark replaces broken image silently             | — (console warning only) |
| Font Loading  | FontFace API in progress       | System fonts shown; `font-display: swap`; then custom fonts | No layout shift          |
| Font Failed   | FontFace.load() rejects        | System fallback fonts persist; colors still apply           | Console warning          |

#### Interactive Elements

| Element      | Type   | Action | Outcome                                     |
| ------------ | ------ | ------ | ------------------------------------------- |
| Tenant Logo  | Button | Click  | Navigate to `/` (home)                      |
| Nav items    | Links  | Click  | Navigate to respective route                |
| [Bell]       | Button | Click  | Opens notification dropdown (Spec 007)      |
| [Globe]      | Button | Click  | Opens language selector dropdown (Spec 006) |
| [AV]         | Button | Click  | Opens user menu dropdown                    |
| [=] (mobile) | Button | Click  | Opens sidebar overlay                       |

#### Responsive Notes

- **1440px**: Full horizontal navigation in header. Logo + text nav + right-aligned actions.
- **768px**: Navigation items may collapse into a "More" dropdown or hamburger. Logo remains visible.
- **375px**: Hamburger menu replaces text nav. Logo centered or left-aligned. Only essential right-side icons shown (avatar).

#### Accessibility

```
Role: banner (header landmark)
Heading hierarchy: N/A (header is navigation, not content heading)
Tab order: [1] Logo (home link) -> [2..N] Nav items -> [N+1] Bell -> [N+2] Globe -> [N+3] Avatar
Focus trap: No

ARIA:
  <header>: role="banner"
  Logo button: aria-label="Go to home page"
  <nav>: role="navigation" aria-label="Main navigation"
  Active nav item: aria-current="page"
  [Bell]: aria-label="Notifications" + aria-describedby (unread count)
  [Globe]: aria-label="Change language"
  [AV]: aria-label="User menu" aria-haspopup="true" aria-expanded="false|true"
  [=] hamburger: aria-label="Open navigation menu" aria-expanded="false|true"

Contrast:
  Nav text on header bg: --foreground / --card = 17.4:1 ✓
  Active nav indicator: --primary on --card = 5.3:1 ✓
  Muted nav text: --muted-foreground / --card = 4.7:1 ✓

Keyboard:
  Tab: moves through all header interactive elements in DOM order
  Enter / Space: activates focused element
  Arrow keys: navigate within dropdown menus (when open)
  Esc: closes open dropdown

Screen reader flow:
  1. "banner"
  2. "link: Go to home page" (logo)
  3. "navigation: Main navigation"
  4. "link: Dashboard" / "link: CRM, current page" / ...
  5. "button: Notifications, 3 unread"
  6. "button: Change language"
  7. "button: User menu"
```

### Screen 5: Widget Loading State

**Source**: FR-023, NFR-015 | **Type**: New

When a widget is lazy-loaded via `loadWidget()`, a loading skeleton is shown
inside the `<Suspense>` boundary while Module Federation fetches the remote

```
| │            │   | ░░░░░░░░░░░░░░░                            |   |
```

### Screen 7: Widget Unavailable Fallback

**Source**: FR-024, Edge Case #8 | **Type**: New

When a widget remote fails to load (plugin not running, network error, or
version mismatch), a static placeholder is shown. Per spec, there is **no
retry button** for widget fallbacks (auto-retries are noisy at widget
granularity).

**Desktop (in context)**:

```
+--------------------------------------------------------------------+
| [Tenant Logo]  Dashboard  CRM  Analytics       [Bell] [Globe] [AV]|
+--------------------------------------------------------------------+
| ┌────────────┐                                                     |
| │  SIDEBAR   │   Contact Detail: Acme Corp                        |
| │            │                                                     |
| │ ...        │   +--------------------------------------------+   |
| │            │   | Name:  John Smith                          |   |
| │            │   | Email: john@acme.com                       |   |
| │            │   | Phone: +1 555-0123                         |   |
| │            │   +--------------------------------------------+   |
| │            │                                                     |
| │            │   Related Widget                                    |
| │            │   +- - - - - - - - - - - - - - - - - - - - - -+   |
| │            │   :                                            :   |
| │            │   :       [box icon]                           :   |
| │            │   :       Widget Unavailable                   :   |
| │            │   :       crm/ContactCard                      :   |
| │            │   :                                            :   |
| │            │   +- - - - - - - - - - - - - - - - - - - - - -+   |
| │            │                                                     |
| └────────────┘                                                     |
+--------------------------------------------------------------------+
```

**Notes**:

- Dashed border (2px, `--border` color) — visually distinct from solid-border cards
- `[box icon]` — `Package` icon from Lucide, `--muted-foreground` color, 24px
- "Widget Unavailable" — `--muted-foreground`, `--font-size-sm`, `--font-medium`
- "crm/ContactCard" — `--muted-foreground`, `--font-size-xs`, `--font-mono`
- No retry button (per spec §9)
- No interactive elements — this is a purely informational placeholder
- Rest of the plugin page remains fully functional

#### State Inventory (Widget — all 3 screens)

| State       | Trigger                     | Visual Change                                | User Feedback                   |
| ----------- | --------------------------- | -------------------------------------------- | ------------------------------- |
| Loading     | `loadWidget()` called       | Skeleton with pulse animation                | aria-busy="true"                |

that should not change per tenant.
