**Source**: FR-013, FR-014 | **Type**: Modified (from flat list to tree)

tree view with expand/collapse, indentation, child count badges, and ARIA tree
roles. This component lives inside the `--sidebar-width: 260px` sidebar.

#### State Inventory

| State     | Trigger                          | Visual Change                                                                | User Feedback                                 |
| --------- | -------------------------------- | ---------------------------------------------------------------------------- | --------------------------------------------- |
| Default   | Sidebar visible                  | Tree with last-expanded state restored from localStorage                     | —                                             |

| Expanded  | Click chevron on parent node     | Children animate in (slide-down, `--transition-normal: 200ms`)               | aria-expanded="true" announced                |

| Element              | Type    | Action                   | Outcome                                         |
| -------------------- | ------- | ------------------------ | ----------------------------------------------- |

#### Responsive Notes

- **1440px**: Sidebar permanently visible at 260px. Full tree with search bar.
- **1024px**: Sidebar permanently visible at 260px. Workspace names may truncate with ellipsis.
- **768px**: Sidebar becomes overlay (280px) triggered by hamburger menu. Backdrop overlay.
- **375px**: Sidebar becomes full-width overlay. Tree items have larger touch targets (44px min height).

#### Accessibility

```
Role: role="tree" on container, role="treeitem" on each node
Heading hierarchy: No headings inside switcher (it's a navigation widget)
Tab order: [1] Switcher button → [2] Search input → [3-N] Tree items (arrow-key nav)
Focus trap: No (dropdown can be dismissed)

ARIA:
  Container: role="tree", aria-label="Workspace hierarchy"
  Each node: role="treeitem", aria-level="{depth+1}", aria-selected="{isActive}"
  Parent nodes: aria-expanded="{isExpanded}", aria-setsize="{siblingCount}", aria-posinset="{index}"
  Badge: aria-label="{count} child workspaces"
  Search: aria-label="Search workspaces", role="searchbox"
  "+ New Workspace": aria-label="Create new workspace"

Contrast (verified against design-system.md §2):
  Workspace name (--foreground) on sidebar (--card): 17.4:1 ✓
  Badge text (--primary-foreground) on badge (--primary): 5.3:1 ✓
  Muted connector lines (--permission-tree-line) on sidebar: decorative, no contrast req
  Selected bg (--muted) sufficient: foreground remains --foreground

Keyboard:
  Enter: Select focused workspace → navigate
  Space: Toggle expand/collapse on parent node
  ArrowDown: Move focus to next visible treeitem
  ArrowUp: Move focus to previous visible treeitem
  ArrowRight: Expand collapsed parent; or move to first child
  ArrowLeft: Collapse expanded parent; or move to parent
  Home: Move focus to first treeitem
  End: Move focus to last visible treeitem
  Esc: Close switcher dropdown

Screen reader flow:
  1. "Workspace hierarchy, tree" (tree container)
  2. "Engineering, expanded, 3 child workspaces, level 1, 1 of 3" (root node)
  3. "Backend, collapsed, 2 child workspaces, level 2, 1 of 3" (child node)
  4. On expand: "expanded" announced
  5. On workspace select: navigation occurs, new page title announced
```

| State           | Trigger                                        | Visual Change                                                   | User Feedback                                                         |
| --------------- | ---------------------------------------------- | --------------------------------------------------------------- | --------------------------------------------------------------------- |

| Slug conflict | Error Alert: "A workspace with this slug exists"  | aria-live="assertive" on error region |
| Success       | Dialog closes; tree re-renders                    | Toast announced via aria-live         |

**Keyboard behavior**:

- `Tab`: Cycle through dialog fields
- `Enter`: Submit (Move Workspace)
- `Esc`: Cancel and close dialog

### Persona-Specific Verification: Aisha (Low Vision)

Aisha uses OS-level zoom at 150% and needs high contrast. Verified:

| Check                                   | Result | Notes                                                           |
| --------------------------------------- | ------ | --------------------------------------------------------------- |
| All text resizable to 200% without loss | ✅     | Relative font sizing (rem); layout reflows at large zoom        |
| Contrast: tree text on bg               | 17.4:1 | `--foreground` on `--background`                                |
| Contrast: badge text on primary         | 5.3:1  | `--primary-foreground` on `--primary`                           |
| Contrast: muted text on bg              | 4.7:1  | `--muted-foreground` on `--background`                          |
| Contrast: error text on card            | 5.8:1  | `--destructive` on `--card`                                     |
| Contrast: warning text on card          | 4.5:1  | `--status-warning` on `--card` (minimum passing)                |
| Focus indicators visible                | ✅     | `--ring` focus ring on all interactive elements                 |
| No color-only status indicators         | ✅     | All badges include text label + icon. Tree uses text + chevron. |
| Screen reader announcements on expand   | ✅     | aria-expanded changes announced                                 |
| Keyboard-only usable                    | ✅     | Full arrow-key tree nav; Tab for form fields; Esc to dismiss    |
