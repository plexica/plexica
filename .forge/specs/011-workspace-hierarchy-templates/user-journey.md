### Journey 1: Priya — View Organizational Hierarchy & Aggregate Data

**Trigger**: Priya opens the Plexica dashboard on Monday morning to review headcount and activity across all engineering teams before a leadership sync.

**Preconditions**:

- Priya is authenticated and has tenant ADMIN role
- Tenant has a 3-level hierarchy: Engineering (root) → Backend, Frontend, QA (children) → Backend has 2 grandchild squads
- Each workspace has members and enabled plugins

#### Happy Path

| Step | User Action                            | System Response                                                                                                                                                      | Emotional State                                 |
| ---- | -------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| 1    | Clicks workspace switcher in sidebar   | Tree view expands showing Engineering (root) with 3 children, one of which has 2 grandchildren. Child count badges show [3] next to Engineering, [2] next to Backend | 😐 Neutral — familiar navigation                |
| 2    | Clicks expand chevron on "Engineering" | Children (Backend, Frontend, QA) appear indented below with member counts                                                                                            | 😊 Satisfied — hierarchy is clear at a glance   |
| 3    | Clicks "Engineering" workspace name    | Navigates to Engineering workspace detail page showing own data plus aggregated descendant counts: "25 members across 6 workspaces, 12 teams"                        | 🎉 Delighted — one-click aggregate view         |
| 4    | Scrolls to children summary section    | Card grid shows each child workspace with name, member count, team count, and child count                                                                            | 😊 Satisfied — no need to click into each child |
| 5    | Clicks "Backend" child card            | Navigates to Backend workspace detail showing Backend data + its 2 grandchild squads                                                                                 | 😐 Neutral — expected drill-down behavior       |

**Outcome**: Priya sees the full organizational structure and aggregate metrics from a single root workspace without switching between 12 workspaces individually.

**Delight opportunities**:

- Aggregated counts update live as she navigates (cached, <2ms)
- Tree view remembers her last expanded/collapsed state across sessions
- Breadcrumb trail shows hierarchy path: Engineering > Backend > Squad Alpha

| 2    | System returns 403                                   | Error page: "You don't have access to this workspace. You can only see workspaces you're a member of. Contact your workspace admin for access." | 😕 Confused → 😐 Informed         |

**Message**: "You don't have access to this workspace. You can only view workspaces where you are a direct member, or descendant workspaces if you are an admin of a parent. Contact your workspace administrator to request access."

### Journey 2: Marco — Create Child Workspace from Template

**Trigger**: Marco needs to spin up a new project workspace ("Project Phoenix") under his "Frontend" team workspace, using the "Engineering Project" template that pre-configures CI/CD, code review, and Kanban plugins.

**Preconditions**:

- Marco is ADMIN of the "Frontend" workspace (depth 1)
- Tenant has the "Engineering Project" template registered by the project-management plugin
- All plugins referenced in the template are installed at tenant level

#### Happy Path

| Step | User Action                                                                                                  | System Response                                                                                                                                                      | Emotional State                                 |
| ---- | ------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| 1    | Clicks "+" button next to "Frontend" in workspace switcher (or uses context menu → "Create Child Workspace") | Create Workspace Dialog opens with "Frontend" pre-selected as parent                                                                                                 | 😊 Convenient — parent auto-filled              |
| 2    | Types "project-phoenix" as slug, "Project Phoenix" as name                                                   | Real-time slug validation shows green checkmark (slug is unique under Frontend)                                                                                      | 😐 Neutral — standard form                      |
| 3    | Sees template picker step: card grid showing available templates + "Blank Workspace" option                  | Cards display: "Engineering Project" (3 plugins, 1 page, 2 settings), "Design Sprint" (2 plugins), "Blank Workspace"                                                 | 🤔 Considering — reviewing options              |
| 4    | Clicks "Engineering Project" template card                                                                   | Card becomes selected (blue border), preview panel shows: "Will configure: CI/CD Pipeline ✓, Code Review ✓, Kanban Board page ✓, Default team role: MEMBER"          | 😊 Confident — sees exactly what will happen    |
| 5    | Clicks "Create Workspace" button                                                                             | Loading state: button shows spinner, progress text "Creating workspace and applying template..."                                                                     | 😐 Waiting — clear feedback                     |
| 6    | Creation succeeds                                                                                            | Success toast: "Project Phoenix created successfully with Engineering Project template". Dialog closes. Workspace switcher updates showing new child under Frontend. | 🎉 Delighted — 30 seconds instead of 15 minutes |
| 7    | Clicks "Project Phoenix" in tree                                                                             | Navigates to new workspace. Kanban Board page exists in nav. CI/CD and Code Review plugins are active in plugin list.                                                | 🎉 Satisfied — everything pre-configured        |

**Outcome**: Marco created a fully configured project workspace in 30 seconds instead of 15 minutes of manual setup. The template applied 3 plugins, 1 page, and 2 settings atomically.

**Delight opportunities**:

- Template card shows plugin icons (Lucide icons matching each plugin)
- Preview panel updates in real-time as template is selected
- After creation, a "View workspace" toast action link takes Marco directly there
