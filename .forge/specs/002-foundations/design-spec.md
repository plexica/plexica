| State           | Trigger                           | Visual Change                                      | User Feedback                                                |

| Empty           | No activity data (new tenant)     | Activity card shows empty state                    | "No recent activity yet. Start exploring your organization." |

#### Interactive Elements

| Element              | Type                    | Action | Outcome                                                          |
| -------------------- | ----------------------- | ------ | ---------------------------------------------------------------- |
| Hamburger (≡)        | button                  | click  | Toggles sidebar (mobile: opens drawer; desktop: collapse/expand) |

#### Responsive Notes

- **1440px**: Sidebar 240px expanded, content max-width 1200px, 4-column stat grid
- **1024px**: Sidebar 240px, content fills remaining width, 2-column stat grid
- **768px**: Sidebar collapsed to 64px (icon-only), content fills width, 2-column stat grid
- **375px**: Sidebar hidden (hamburger to open as overlay drawer), 1-column stat grid, full-width cards

#### Accessibility

```

Contrast:
  Body text on card: neutral-700 / #FFFFFF = 11.0:1 ✓
  Welcome heading: neutral-900 / neutral-50 = 18.4:1 ✓
  Sidebar nav item: neutral-600 / neutral-100 = 5.0:1 ✓
  Active nav item: primary-600 / primary-50 = 5.9:1 ✓
  Stat value: neutral-900 / #FFFFFF = 18.4:1 ✓

Keyboard:
  Tab: moves through focusable elements in logical order
  Enter/Space: activates buttons and links
  Esc: closes sidebar drawer (mobile), closes user menu dropdown
  Arrow Up/Down: navigates within user menu dropdown
  Home/End: first/last item in user menu

Screen reader flow:
  1. "Skip to main content, link"
  2. "Plexica, banner"
  3. "Main navigation, navigation landmark"
  4. "Dashboard, current page, link" (active nav)
  5. "Main content, main landmark"
  6. "Welcome back, Maria, heading level 1"
  7. "Quick Stats, heading level 2"
  8. [Stat card content]
  9. "Recent Activity, heading level 2"
```

### Screen 5: Session Expired State

**Source**: 002-04, 002-05, EC-05 | **Type**: New (overlay on existing shell)

This is not a standalone page — it's a toast notification + redirect behavior
triggered when the refresh token fails.

```
┌──────────────────────────────────────────────────────────────────┐
│ (Existing shell — grayed out / inactive)                         │
│                                                                  │
│  ┌──────────────────────────────────────────┐  ← Toast (top-right)
│  │ ⚠ Your session has expired.              │                    │
│  │   Redirecting to sign-in…                │                    │
│  │                                   [✕]    │                    │
│  └──────────────────────────────────────────┘                    │
│                                                                  │
│  (After 3 seconds: full redirect to Keycloak login)              │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

#### State Inventory

| State                 | Trigger                      | Visual Change                               | User Feedback                                       |
| --------------------- | ---------------------------- | ------------------------------------------- | --------------------------------------------------- |
| Token refresh attempt | API returns 401              | No visible change (happens in background)   | —                                                   |
| Refresh failed        | Refresh token exchange fails | Warning toast appears; all API calls paused | "Your session has expired. Redirecting to sign-in…" |
| Redirect              | 3s after toast               | Full page redirect to Keycloak              | Keycloak login page loads                           |

#### Interactive Elements

| Element           | Type   | Action | Outcome                                  |
| ----------------- | ------ | ------ | ---------------------------------------- |
| Toast dismiss (✕) | button | click  | Dismisses toast (redirect still happens) |

#### Accessibility

```
ARIA:
  [toast]: role="alert", aria-live="assertive"
  [dismiss button]: aria-label="Dismiss notification"

Contrast:
  Warning text: warning-dark (#B45309) / warning-light (#FEF3C7) = 5.1:1 ✓

Screen reader flow:
  1. "Alert: Your session has expired. Redirecting to sign-in."
```

### Screen 6: Error Boundary Fallback

**Source**: 002-16 | **Type**: New

Renders inside the content area when a route-level component throws. The
shell (sidebar + header) remains functional.

```
+------+-----------------------------------------------------------+
|      |                                                           |
| SIDE |  ┌──────────────────────────────────────────────┐        |
| BAR  |  │                                              │        |
| (ok) |  │        [Icon: AlertTriangle (Lucide)]        │        |
|      |  │                                              │        |
|      |  │        Something went wrong                  │  ← h1  |
|      |  │                                              │        |
|      |  │        We hit an unexpected error.            │        |
|      |  │        You can try refreshing this page       │        |
|      |  │        or return to the dashboard.            │        |
|      |  │                                              │        |
|      |  │     [ Go to Dashboard ]   [ Refresh ]        │        |
|      |  │       (primary)            (secondary)       │        |
|      |  │                                              │        |
|      |  └──────────────────────────────────────────────┘        |
|      |                                                           |
+------+-----------------------------------------------------------+
```

#### State Inventory

| State                | Trigger                             | Visual Change                       | User Feedback          |
| -------------------- | ----------------------------------- | ----------------------------------- | ---------------------- |
| Error caught         | JS runtime error in route component | Content area replaced with fallback | "Something went wrong" |
| Recovery (dashboard) | User clicks "Go to Dashboard"       | Navigate to `/`                     | Dashboard loads        |
| Recovery (refresh)   | User clicks "Refresh"               | `window.location.reload()`          | Page refreshes         |

#### Interactive Elements

| Element         | Type             | Action | Outcome                         |
| --------------- | ---------------- | ------ | ------------------------------- |
| Go to Dashboard | primary button   | click  | TanStack Router navigate to `/` |
| Refresh         | secondary button | click  | Full page reload                |

#### Accessibility

```
Role: main (replaces normal content)
Heading hierarchy: h1 "Something went wrong"
Tab order: [1] Go to Dashboard → [2] Refresh
Focus trap: no

ARIA:
  [AlertTriangle icon]: aria-hidden="true"
  [error region]: role="alert"

Contrast:
  Heading: neutral-900 / neutral-50 = 18.4:1 ✓
  Body: neutral-600 / neutral-50 = 7.0:1 ✓
  Primary button: #FFFFFF / primary-500 = 4.7:1 ✓

Keyboard:
  Tab: Go to Dashboard → Refresh
  Enter/Space: activates focused button

Screen reader flow:
  1. "Alert: Something went wrong"
  2. "We hit an unexpected error..."
  3. "Go to Dashboard, button"
  4. "Refresh, button"
```
