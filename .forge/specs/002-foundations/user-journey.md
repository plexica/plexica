## 2. User Journeys

---

### Journey 1: Maria — Login to Tenant Dashboard

**Trigger**: Maria opens her browser and navigates to `acme.plexica.io` to
start her work day. Per 002-06 (realm discovery from subdomain).

**Preconditions**:

- Maria has valid Keycloak credentials in the `tenant-acme` realm
- The `acme` tenant is provisioned and active
- No existing session (first login of the day)

#### Happy Path

| Step | User Action                                                     | System Response                                                                                                                     | Emotional State                |
| ---- | --------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- | ------------------------------ |
| 1    | Types `acme.plexica.io` in browser                              | App shell loads, detects `acme` subdomain (per 002-06), redirects to Keycloak `tenant-acme` realm login page                        | 😐 Neutral — "Here we go"      |
| 2    | Enters email and password on Keycloak login page                | Keycloak validates credentials, issues RS256 JWT (per 002-02), redirects back to app with auth code                                 | 😐 Neutral — waiting           |
| 3    | Lands on app; auth code exchanged for tokens                    | App stores tokens in Zustand auth store (per 002-18), TanStack Query fetches user profile                                           | 😊 Relieved — "I'm in"         |
| 4    | Sees dashboard with personalized welcome: "Welcome back, Maria" | Shell layout renders with sidebar, header, content area (per 002-13). Dashboard shows welcome message with user's name (per 002-15) | 🎉 Satisfied — "Ready to work" |

**Outcome**: Maria is authenticated in her tenant's realm, sees her name on
the dashboard, and can navigate the app shell. Entire flow < 3s (per NFR-01).

**Delight opportunities**:

- Show a subtle loading skeleton during auth redirect to avoid blank screens
- Animate the welcome message on first load of the day
- Pre-warm TanStack Query cache for frequently accessed data

#### Edge Case A: Unknown Tenant Subdomain

**Trigger**: Maria mistypes the subdomain or uses a bookmark from a
decommissioned tenant. Per EC-01 / EC-02.

| Step | User Action                               | System Response                                                                                                                                                    | Emotional State                        |
| ---- | ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------- |
| 1    | Types `amce.plexica.io` (typo) in browser | App detects subdomain `amce`                                                                                                                                       | 😐 Neutral                             |
| 2    | —                                         | System cannot resolve tenant. Displays error page: "We couldn't find this organization. Check the URL and try again." No reveal of valid tenant slugs (per EC-02). | 😤 Frustrated — "Did I type it wrong?" |
| 3    | Checks URL, corrects to `acme.plexica.io` | Normal login flow resumes                                                                                                                                          | 😊 Relieved                            |

**Recovery path**: Clear error message with suggestion to check the URL.
Contact admin link provided for persistent issues.

**Feedback**:

- Heading: "Organization not found"
- Body: "We couldn't find an organization at this address. Double-check the URL or contact your administrator."
- No CTA that lists valid tenants (security: prevents tenant enumeration).

---

#### Edge Case B: Session Expired During Work

**Trigger**: Maria's session exceeds the max lifetime (Keycloak SSO session,
typically 8 hours). Refresh token is no longer valid. Per EC-05 / 002-05.

| Step | User Action                                | System Response                                                                                                                                   | Emotional State                                    |
| ---- | ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------- |
| 1    | Clicks a navigation item or submits a form | API returns 401; auth store attempts refresh token exchange                                                                                       | 😐 Unaware                                         |
| 2    | —                                          | Refresh fails (max session exceeded). Auth store clears tokens (per EC-05). Toast notification: "Your session has expired. Please sign in again." | 😤 Frustrated — "I was in the middle of something" |
| 3    | Is redirected to Keycloak login page       | Keycloak login page loads for `tenant-acme` realm                                                                                                 | 😐 Neutral — knows what to do                      |
| 4    | Re-enters credentials                      | Normal login flow; returns to last-visited route (if supported)                                                                                   | 😊 Relieved                                        |

**Recovery path**: Automatic redirect to login. Unsaved form data is lost
(accepted trade-off for Phase 1 — form recovery deferred).

**Feedback**:

- Toast (warning variant): "Your session has expired. Please sign in again."
- Toast persists for 5 seconds, then auto-dismisses.
- No sensitive information in the message.

---

#### Edge Case C: Authentication Failure

**Trigger**: Maria enters wrong credentials on the Keycloak login page.

| Step | User Action                            | System Response                                                                                                                | Emotional State                                        |
| ---- | -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------ |
| 1    | Enters wrong password on Keycloak page | Keycloak shows inline error: "Invalid username or password."                                                                   | 😤 Frustrated                                          |
| 2    | Retries with correct password          | Normal flow continues                                                                                                          | 😊 Relieved                                            |
| 3    | (Alt) Exceeds max login attempts       | Keycloak locks account per realm brute-force policy (per NFR-09). Shows: "Account temporarily locked. Try again in N minutes." | 😡 Frustrated — but this is expected security behavior |

**Recovery path**: Keycloak-managed. Error messages come from Keycloak theme.
Platform does not customize Keycloak login error messages in Phase 1.

### Journey 2: Sam — Daily Login and Dashboard Navigation

**Trigger**: Sam clicks a bookmark or types `acme.plexica.io` on their
mobile phone during commute, or on desktop at work.

**Preconditions**:

- Sam has valid credentials in the `tenant-acme` realm
- Sam is a regular tenant member (not admin)

#### Happy Path

| Step | User Action                           | System Response                                                                                                | Emotional State                           |
| ---- | ------------------------------------- | -------------------------------------------------------------------------------------------------------------- | ----------------------------------------- |
| 1    | Opens `acme.plexica.io`               | Shell loads with loading skeleton (per NFR-07, FCP < 1.5s)                                                     | 😐 Neutral                                |
| 2    | Redirected to Keycloak login          | Keycloak login page renders for `tenant-acme`                                                                  | 😐 Neutral                                |
| 3    | Enters credentials                    | Token exchange, redirect back to app                                                                           | 😐 Waiting                                |
| 4    | Dashboard loads: "Welcome, Sam"       | Shell layout: sidebar with nav items, header with avatar and logout, content area with welcome (per 002-13/15) | 🎉 Satisfied — "I'm in, this looks clear" |
| 5    | Uses keyboard Tab to navigate sidebar | Focus indicators visible on all nav items (per 002-20), logical tab order                                      | 😊 Confident — "I can navigate this"      |

**Outcome**: Sam authenticated, personalized dashboard visible, all
interactive elements keyboard-navigable with visible focus indicators.

**Delight opportunities**:

- Large, high-contrast text for Sam's low-vision needs
- Tab key navigates through sidebar items predictably
- Skip-to-content link for keyboard users

---

#### Edge Case A: Login Page with No Subdomain

**Trigger**: Sam navigates to `plexica.io` directly (no subdomain). Per EC-01.

| Step | User Action        | System Response                                                                                                                              | Emotional State                |
| ---- | ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------ |
| 1    | Opens `plexica.io` | No tenant slug detected                                                                                                                      | 😐 Neutral                     |
| 2    | —                  | Error page: "Which organization are you looking for? Enter your organization URL (e.g., your-org.plexica.io) or contact your administrator." | 😕 Confused — "Where do I go?" |

**Recovery path**: Error page with guidance. No login form rendered (no
realm to authenticate against).

**Feedback**:

- Heading: "Which organization are you looking for?"
- Body: "Enter your organization address in the browser. It looks like: your-org.plexica.io"
- Help link: "Contact your administrator if you don't know your organization address."

---

#### Edge Case B: Runtime Error in App (Error Boundary)

**Trigger**: A component throws an unhandled error after Sam is logged in.
Per 002-16.

| Step | User Action                                   | System Response                                                                                                                            | Emotional State                                     |
| ---- | --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------- |
| 1    | Navigates to a section that throws a JS error | Route-level error boundary catches the error                                                                                               | 😐 Unaware                                          |
| 2    | —                                             | Error boundary renders: "Something went wrong. Try refreshing the page or go back to the dashboard." Sidebar and header remain functional. | 😤 Mildly frustrated                                |
| 3    | Clicks "Go to Dashboard" link or refreshes    | Dashboard loads normally                                                                                                                   | 😊 Relieved — "At least the whole app didn't break" |

**Recovery path**: Error boundary is scoped to route content area. Navigation
shell (sidebar + header) remains functional. User can navigate away.

**Feedback**:

- Heading: "Something went wrong"
- Body: "We hit an unexpected error. You can try refreshing this page or return to the dashboard."
- CTA: "Go to Dashboard" (link) | "Refresh" (button)
