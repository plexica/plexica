# ADR-010: Keycloakify for Keycloak Custom Theme

**Date**: April 2026
**Status**: Accepted
**Deciders**: Platform team

## Context

Plexica uses Keycloak multi-realm for authentication. The default Keycloak login UI does not match the Plexica design system (@plexica/ui). Users see a generic Keycloak form instead of a branded, consistent experience.

## Decision

Use Keycloakify (v11) to build the Keycloak theme as a React/Vite project. Keycloakify compiles React components into a valid Keycloak theme JAR deployed in the Keycloak `providers/` directory.

## Rationale

- Reuse @plexica/ui tokens (CSS custom properties) directly — no manual CSS duplication
- Type-safe page components: Keycloakify generates TypeScript types for each Keycloak page
- Hot-reload development with `--mock` mode (no Keycloak required during UI development)
- Compatible with Keycloak 26+

## Build & Deployment

The build pipeline is:

```
pnpm --filter @plexica/keycloak-theme build
```

This runs:

1. `vite build` — compiles React theme into `dist/`
2. `keycloakify build` — packages into `dist_keycloak/keycloak-theme-for-kc-all-other-versions.jar`
3. Copies JAR to `infra/keycloak/providers/plexica-theme.jar`

Docker Compose mounts `./infra/keycloak/providers:/opt/keycloak/providers:ro`. Keycloak 26+ auto-discovers JARs in `providers/` at startup (no manual JAR installation required).

> **Note**: Keycloak 26+ requires JARs in `providers/`, not the legacy `themes/` directory. The
> `infra/keycloak/themes/` directory is not used. The JAR file is excluded from git via
> `.gitignore`; it is regenerated by the build script before container startup.

## Font Strategy

Inter is self-hosted via `@fontsource/inter` — no external Google Fonts request. This avoids GDPR
concerns with external font loading inside the Keycloak iframe context.

## Alternatives Considered

- **Vanilla FreeMarker theme**: HTML templates + manual CSS. No React reuse, manual token sync required. Rejected.
- **Transparent redirect (SPA login screen)**: Complex, insecure, incompatible with multi-realm SAML/MFA flows. Rejected.
- **Directory mount (no JAR)**: Keycloakify v11 always produces a JAR as its primary output. Directory mount is not supported in this version. Rejected.

## Consequences

- New dependency: `keycloakify` (v11), `@fontsource/inter` in `packages/keycloak-theme`
- New package `packages/keycloak-theme` added to pnpm workspaces
- Docker Compose updated: `providers/` mount replaces the old `themes/` mount
- Build script copies JAR to `infra/keycloak/providers/plexica-theme.jar`
- Theme name registered in realm config as `plexica` (`loginTheme: "plexica"`)
- JAR artifact added to `.gitignore` — regenerated at build time
